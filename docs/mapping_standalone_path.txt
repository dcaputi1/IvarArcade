# COMPLETE BUTTON MAPPING - PHYSICAL TO MAME STANDALONE (SA)
# This documents the Standalone (sa) path: Physical → Joystick → MAME Direct

## STANDALONE MAME INPUT SYSTEM
Standalone MAME reads joystick inputs directly without a RetroPad layer.
Button mappings are controlled by /opt/retropie/emulators/mame/cfg_sa/default.cfg

## KEY DIFFERENCES: cfg_ra vs cfg_sa
================================================================================

1. **Device Usage:**
   - cfg_ra: Uses JOYCODE_1/2/3/4 (Trooper, Xinmo, Ultra, Xbox)
   - cfg_sa: Uses JOYCODE_2/3/4/5 (no JOYCODE_1 Trooper in default!)

2. **Button ID Mapping:**
   - cfg_ra: Generally sequential button IDs
   - cfg_sa: Non-sequential, custom mappings per device

3. **Directional Controls:**
   - cfg_ra: Prefers HAT inputs (HAT1UP/DOWN/LEFT/RIGHT)
   - cfg_sa: Prefers AXIS inputs (YAXIS/XAXIS_UP/DOWN/LEFT/RIGHT_SWITCH)

4. **UI Controls:**
   - cfg_ra: Simple UI_SELECT/UI_BACK (JOYCODE_1_BUTTON2/BUTTON1)
   - cfg_sa: Complex UI mappings across multiple devices with keyboard fallbacks

5. **Special Mappings:**
   - Both: START 2 button on js3 maps to Player 2 start
   - cfg_sa: Uses JOYCODE_1_BUTTON7 reference even though JOYCODE_1 not in default
   - cfg_sa: More keyboard fallbacks (KEYCODE_1, KEYCODE_5, KEYCODE_Z, KEYCODE_X)

6. **Button Count Usage:**
   - cfg_ra: Uses more standard button IDs (1-6 mainly)
   - cfg_sa: Uses higher button IDs (9, 10, 11, 12) for some controls

================================================================================
## RECOMMENDED USAGE
================================================================================

For games requiring precise control mapping:
- Use cfg_ra (RetroArch path) for consistent retropad-based mapping
- Easier to reconfigure through RetroArch's input settings

For games requiring direct hardware access:
- Use cfg_sa (Standalone path) for direct MAME input
- More complex but allows game-specific .cfg files per ROM
