## QBERT - Special Diagonal Controls
================================================================================

Q*bert requires diagonal movement (isometric gameplay), so the joystick is
remapped to support 4-way diagonal input instead of standard 4-way cardinal.

### cfg_ra (RetroArch) - Using JOYCODE_1 (likely Trooper V2):

Movement          | Input Combination
------------------|------------------------------------------
P1_JOYSTICK_RIGHT | HAT1DOWN + HAT1RIGHT (down-right diagonal)
P1_JOYSTICK_LEFT  | HAT1UP + HAT1LEFT (up-left diagonal)
P1_JOYSTICK_UP    | HAT1UP + HAT1RIGHT (up-right diagonal)
P1_JOYSTICK_DOWN  | HAT1DOWN + HAT1LEFT (down-left diagonal)

P2_JOYSTICK_RIGHT | HAT1RIGHT + HAT1DOWN
P2_JOYSTICK_LEFT  | HAT1UP + HAT1LEFT
P2_JOYSTICK_UP    | HAT1UP + HAT1RIGHT
P2_JOYSTICK_DOWN  | HAT1DOWN + HAT1LEFT

Special Mappings:
- START2: KEYCODE_2 OR JOYCODE_5_START OR JOYCODE_1_BUTTON5
- COIN1: KEYCODE_5 OR JOYCODE_1_SELECT OR JOYCODE_2_SELECT OR JOYCODE_3_SELECT 
         OR JOYCODE_4_SELECT OR JOYCODE_1_BUTTON2

### cfg_sa (Standalone) - Using JOYCODE_1:

Movement          | Input Combination
------------------|--------------------------------------------------
P1_JOYSTICK_RIGHT | XAXIS_RIGHT_SWITCH + YAXIS_DOWN_SWITCH (right-down)
P1_JOYSTICK_LEFT  | XAXIS_LEFT_SWITCH + YAXIS_UP_SWITCH (left-up)
P1_JOYSTICK_UP    | XAXIS_RIGHT_SWITCH + YAXIS_UP_SWITCH (right-up)
P1_JOYSTICK_DOWN  | XAXIS_LEFT_SWITCH + YAXIS_DOWN_SWITCH (left-down)

P2_JOYSTICK_RIGHT | YAXIS_DOWN_SWITCH + XAXIS_RIGHT_SWITCH
P2_JOYSTICK_LEFT  | YAXIS_UP_SWITCH + XAXIS_LEFT_SWITCH
P2_JOYSTICK_UP    | YAXIS_UP_SWITCH + XAXIS_RIGHT_SWITCH
P2_JOYSTICK_DOWN  | XAXIS_LEFT_SWITCH + YAXIS_DOWN_SWITCH

Notes:
- Both P1 and P2 use same joystick (JOYCODE_1) for Q*bert
- Diagonal mapping essential for Q*bert isometric gameplay
- cfg_ra uses HAT inputs, cfg_sa uses AXIS inputs
- Coins recorded: 6 (in cfg_ra)

## KEY OBSERVATIONS - Custom Game Configs
================================================================================

1. **Diagonal Movement Games (Q*bert):**
   - Require AND logic (two directions simultaneously)
   - cfg_ra uses HAT combinations
   - cfg_sa uses AXIS combinations

2. **Analog Control Games (Spy Hunter):**
   - Use AXIS inputs for steering/pedals
   - XAXIS = steering wheel
   - RYAXIS/ZAXIS = pedals
   - Mix of digital buttons + analog controls

3. **Multi-Controller Games (Indianapolis Tempest):**
   - Accept input from multiple JOYCODE devices
   - OR logic allows any controller to trigger action
   - Useful for arcade cabinets with multiple control panels

4. **Difference Between cfg_ra and cfg_sa:**
   - cfg_ra: More complex, includes multiple controller fallbacks
   - cfg_sa: Often simpler, single-controller focus
   - cfg_ra: Tends to use HAT inputs
   - cfg_sa: Tends to use AXIS inputs

5. **Game-Specific Overrides:**
   - Only override necessary controls, inherit rest from default.cfg
   - Can include audio settings (master volume, effects)
   - Can include video settings (views, overlays)
   - Track coin insertion counts (counters)

================================================================================
## USAGE RECOMMENDATIONS
================================================================================

When creating custom game configs:

1. Start with default.cfg as base
2. Only override controls that differ from standard layout
3. For diagonal games: Use AND logic (space-separated in newseq)
4. For multi-input: Use OR logic (OR keyword in newseq)
5. Test both cfg_ra and cfg_sa versions
6. Consider analog vs digital control requirements
7. Document special requirements (spinners, trackballs, etc.)
